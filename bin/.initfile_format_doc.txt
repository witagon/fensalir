Format of FILE is line-based and each line consists of the fields

OS SOURCE OWNER VERSION URI TYPE BUILD-SYSTEM DEPS TOOLS RELEASE-LOCATION

where

OS                Operating System line is valid for
SOURCE            Archive or Version Control System used for repo, only valid
                    value is 'git'
OWNER             Organizational entity owning the repo/archive
VERSION           Version tag or keyword 'floating'; floating means that HEAD on
                    selected branch is used. Branch is automatically selected
                    based on JIRA issue id, unless a tag-name is used.
URI               URI to repo/archive as used by VCS/archive tool when
                    retrieving repo/archive from server
TYPE              Type of repo/archive; allowed values are
                    'app' or 'library'
BUILD-SYSTEM      Kind of build system to use when building the repo;
                    2-tuple field, see below for details
DEPS              Dependencies to other repos/archives;
                    list of 3-tuples, see below for details
TOOLS             List of tools to use when building, see below for details
RELEASE-LOCATION  3-tuple (country, site, and domain) where releases are allowed
                    to be created

Note: Tuples use ';' as separator between fields and lists use ',' as separator
      between elements of the list. Reserved keyword 'none' is used for a blank
      field.


BUILD-SYSTEM 2-tuple (or keyword 'none') with type of build system
('msbuild' or 'cmake') followed by type of input file. For MSBuild the
types of supported input files are 'sln' or 'csproj'. For CMake the
only supported input file type is 'cmake'.
